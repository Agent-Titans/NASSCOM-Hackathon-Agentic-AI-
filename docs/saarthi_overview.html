SAARTHI

Intelligent IT service management — classify every incident, route to the right team, and deliver resolution guidance in seconds.

What problem we solve

Enterprise IT helpdesks receive thousands of heterogeneous tickets — password resets, VPN failures, database outages, security incidents. Manual triage is slow, inconsistent, and expensive.

SAARTHI automates the first mile: every ticket is sanitized, classified, matched against 1,000 resolved historical cases, routed to the correct department queue, and given actionable resolution steps — before a human agent opens the inbox.

Three Hands — how work is delivered

1
Self-Help

Employee gets guided steps on the portal. Marks Worked or Did not work to close or escalate.

2
Team Assist

Ticket lands in the right department queue. Agent assigns, collaborates, and resolves.

3
Specialist

Confirmed security incidents route to SecOps. No self-help steps — human specialist required.

End-to-end flow

Employee submits ticket (title + description) │ ▼ Guardrail — redact PII, block injection │ ▼ RAG retrieval — search 1,000 resolved tickets (ChromaDB) │ ▼ Classifier — category + confidence │ ▼ Router — Hand 1/2/3 + department queue │ ▼ Resolver — steps from similar case or playbook │ ▼ Supervisor — final hand + confidence band │ ▼ Employee portal / Agent queue / SecOps desk

SecOps

Routing Specialists desk

Key features

  • RAG grounding — 1,000 resolved synthetic tickets indexed in ChromaDB; similarity gate prevents weak matches
  • Confidence scoring — every ticket stores c_total; UI shows Strong / Good / Low match labels
  • Guardrail layer — PII redaction and prompt-injection defence before any model call
  • Routing Specialists — human-in-the-loop misroute correction without re-running the pipeline
  • Audit trail — every agent step, RAG hit/miss, and duration logged for compliance
  • Reference articles — agents and employees can open similar resolved cases as guidance
  • Multi-firm scenarios — tested across Microsoft, Google, Wipro, and Nasscom ticket patterns
  • Responsible AI — security incidents force Hand 3; low-grounding flag when RAG match is weak

RAG corpus

SAARTHI ships with 1,000 resolved historical tickets (`data/synthetic/tickets_1000.json`). Each title includes a demo assignee name so retrieval results look like real enterprise data.

On first run, bootstrap_rag_environment.py embeds the corpus into ChromaDB (~5–8 minutes). After that, retrieval is local and fast.
