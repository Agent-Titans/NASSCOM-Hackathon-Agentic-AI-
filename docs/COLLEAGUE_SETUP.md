# Colleague setup — SAARTHI (Use Case 1)

**Goal:** Clone → embeddings + RAG ready → run the app → test with demo accounts.  
**Time:** ~10 minutes first run (Gemini batch embed for 1006 corpus vectors).

---

## What you need

| Item | Notes |
|------|--------|
| Python **3.10+** | 3.11 recommended |
| **Google Gemini API key** | Classifier + Resolver + **RAG embeddings** ([aistudio.google.com](https://aistudio.google.com)) |
| **Embeddings** | `gemini-embedding-001` via batch API (~5–8 min first ingest, cached after) |

---

## Quick start (copy-paste)

```bash
git clone <repo-url>
cd "Final Round Hackathon"

# 1. Virtual environment
bash scripts/setup_venv.sh
source .venv/bin/activate

# 2. Environment file
cp .env.example .env
# Edit .env — add GOOGLE_API_KEY=your_key_here
# RAG_EMBEDDING_BACKEND=gemini (default — same vectors as routing test reports)

# 3. Stop any old Streamlit, then bootstrap RAG (one command)
python scripts/bootstrap_rag_environment.py

# 4. Run app
bash scripts/run_app.sh
```

Open **http://localhost:8501** · Password for all accounts: **`1234`**

---

## What bootstrap does

1. Clears **live** UI tickets (keeps demo users)
2. **Patches 1000 RAG titles** with demo assignee names (Sree, Subbu, Narsimha, etc.)
3. Builds **ChromaDB** vector index from `data/synthetic/tickets_1000.json`
4. Loads **1,000 RESOLVED** historical tickets into SQLite (`syn-*` rows)
5. Exports **`data/synthetic/tickets_1000.csv`** for Excel review
6. Uses **Gemini** embeddings (`gemini-embedding-001`, batch API)

**Verify:**

```bash
python -c "
from src.stores.chroma_store import ChromaTicketStore
from src.db.session import get_session_factory, init_db
from src.db.models import Ticket
init_db()
s = get_session_factory()()
print('Chroma docs:', ChromaTicketStore().count)
print('SQLite syn-*:', s.query(Ticket).filter(Ticket.ticket_id.like('syn-%')).count())
"
```

Expected: **Chroma ≈ 1006** · **SQLite syn-* = 1000**

Optional smoke test:

```bash
python scripts/ingest_synthetic_corpus.py --smoke
```

## RAG corpus files (readable)

| File | Purpose |
|------|---------|
| `data/synthetic/tickets_1000.json` | Source corpus — **1000 titles include assignee names** |
| `data/synthetic/tickets_1000.csv` | Same data in Excel-friendly CSV (regenerated by bootstrap) |
| `data/chroma/` | Vector index (**gitignored** — built locally) |

**Manual refresh** (after pulling corpus changes):

```bash
# Stop Streamlit first (avoids SQLite lock)
python scripts/patch_rag_assignee_titles.py   # ensure assignee names in titles
python scripts/ingest_synthetic_corpus.py     # re-embed into Chroma
python scripts/export_synthetic_corpus_csv.py # refresh CSV
```

**Do not** run `generate_synthetic_corpus.py` unless you intend to overwrite the committed 1k corpus.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `database is locked` | Stop Streamlit (`Ctrl+C`), re-run bootstrap |
| Chroma count 0 | `python scripts/ingest_synthetic_corpus.py` |
| Titles missing assignee names | `python scripts/patch_rag_assignee_titles.py` then re-ingest |
| Slow first ingest | Normal — Gemini batch embed ~5–8 min |
| Gemini 503 | Wait 30s, retry; free tier rate limits |
| Wrong logins | Use emails above, password `1234` |
| Email not sending | Check `EMAIL_NOTIFICATIONS_ENABLED=true` and Gmail app password |
